Version History

Each section of the style guide is independently versioned and adheres to semantic versioning in the format major.minor.patch.

  • Major release: Includes changes to specifications that are incompatible with existing specifications.
  • Minor release: Adds new specifications that are backwards-compatible.
  • Patch release: Includes bug fixes and/or minor specification clarifications.
